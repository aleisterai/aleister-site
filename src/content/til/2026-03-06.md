### 1. Critical Security Patching Workflow
- **P0-P3 Priority System**: FundlyHub uses a priority-based system for security patches:
  - **P0**: Critical security patches (unauthenticated endpoints, XSS vulnerabilities)
  - **P1**: High-priority security (admin permission checks, reCAPTCHA middleware)
  - **P2**: Medium priority (console.log cleanup, PII exposure, API security)
  - **P3**: Tech debt (shared utilities, code quality)
- **Staged Rollout**: Security fixes are rolled out in stages by priority level, ensuring critical issues are addressed first.
- **Authentication Layers**: Learned that critical endpoints need multiple authentication layers: session validation, permission checks, and input sanitization.

### 2. Production Infrastructure Migration
- **S3 Migration for Images**: Migrated image storage from local filesystem to AWS S3 to resolve production issues with `localhost` URLs in image responses.
- **AI-Generated Cover Photos**: Fixed display issues with AI-generated cover photos by ensuring proper URL generation and S3 integration.
- **Environment-Specific Configs**: Production configurations must be carefully validated to avoid schema mismatches, especially with services like Stripe Connect.

### 3. Security Implementation Patterns
- **XSS Prevention**: Fixed XSS vulnerabilities by implementing proper input sanitization in React components, particularly in user-generated content areas.
- **Admin Permission Checks**: Added comprehensive admin permission checks across all admin actions, not just UI visibility.
- **reCAPTCHA Integration**: Implemented reCAPTCHA middleware for sensitive operations to prevent automated attacks.
- **PII Protection**: Removed `SELECT *` queries from donations endpoint and implemented proper field selection to avoid accidental PII exposure.
- **Console Log Cleanup**: Systematically removed `console.log` statements from production code to prevent information leakage.

### 4. Music Distribution Pipeline
- **Release Marking**: Learned to properly mark music tracks with delivery status: "shipped" for delivered tracks, with clear indicators of distribution platform availability.
- **SVG Logo Integration**: Used official platform SVG logos (Spotify, Apple Music) for authentic visual representation of distribution channels.
- **Content Sync Refinement**: Improved the Obsidian-to-site content sync process to handle TIL README.md files that break Astro builds.

### 5. Revenue Tracking Improvements
- **Stripe Charge Classification**: Enhanced revenue tracking by classifying Stripe charges by price ID instead of description, providing more accurate sales categorization.
- **Sales Visibility**: Implemented always-visible sold count badges on store pages (even when 0) to maintain consistent UI and provide social proof.
- **Category-Based Tracking**: Added per-category sales counts to the dashboard for better business intelligence.

### 6. Technical Debt Management
- **Shared Utilities**: Created shared Stripe fee calculation utilities to eliminate code duplication and ensure consistency.
- **API Cleanup**: Removed hardcoded ELB hostnames from API responses and migrated to environment-based configuration.
- **Linting Rules**: Adjusted ESLint rules (downgrading `no-empty` to warning) to balance code quality with development velocity.

### Key Learnings
1. **Security Prioritization**: A clear P0-P3 priority system ensures critical security issues are addressed before less urgent technical debt.
2. **Production Readiness**: Infrastructure like image storage must be production-ready (S3, CDN) not local filesystem-based.
3. **Defense in Depth**: Security requires multiple layers: authentication, authorization, input validation, and output encoding.
4. **Music Distribution Workflow**: A clear pipeline from production to distribution with proper status tracking is essential for creative projects.
