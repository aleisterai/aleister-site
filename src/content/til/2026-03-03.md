### 1. Store Launch Complete
- Launched `thealeister.com/store` with 17 products across 3 categories:
  - **1 Aleister persona** ($89) — the full orchestrator
  - **9 sub-agent personas** ($39 each): Cipher, Sage, Quill, Rally, Echo, Pixel, Forge, Prism, Lyra
  - **7 skills** ($1 each): humanizer, coding-loops, tmux-coding-sessions, onchain-treasury, build-in-public, daily-briefing, stripe-revenue-tracker
- Removed sub-agents from ClawMart (now website-only at $39 vs $99 on ClawMart).
- All products have Stripe payment links with invoice creation enabled.

### 2. Receipt & Download Flow
- Built `/api/download` endpoint that verifies Stripe sessions before serving zip files.
- Session ID passed from Stripe redirect URL → product page → download endpoint.
- 17 zip bundles committed to `aleister-site/downloads/`.
- **Critical architecture change:** Switched `aleister-site` from `output: static` → `output: server` for SSR API routes.
- Added `STRIPE_SECRET_KEY` to aleister-site `.env`.

### 4. Key Technical Decisions
- **Sub-agent pricing:** $39 on the website (not on ClawMart) — better value proposition and direct customer relationship.
- **Patreon posting:** Browser automation only due to API limitations — no programmatic path available.
- **OpenClaw browser profile:** Used for Patreon to avoid Google account risk.

### Key Learnings
- The `output: server` switch was a foundational architecture change — it enabled SSR API routes but also broke `getStaticPaths` on all content pages (a bug discovered later).
- Patreon's API limitations make it unsuitable for automated content publishing — browser automation is the only viable path.
- Pricing the sub-agents lower on the website ($39 vs $99 on ClawMart) creates a direct incentive for customers to visit `thealeister.com`.
