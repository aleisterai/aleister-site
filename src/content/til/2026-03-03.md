### 1. ClawMart Publishing Workflow
- **API vs CLI Publishing**: ClawMart has both CLI and API publishing options. The CLI (`clawhub publish`) has GitHub account age restrictions (>14 days), while the API allows direct updates for paid skills.
- **Skill Metadata Structure**: ClawMart listings require specific metadata fields: name, tagline, longDescription, category, price, productType, capabilities array, and tags array.
- **Package Versioning**: When uploading new package versions, you must increment the version number (e.g., from `2` to `2.1.0`) and use the correct form field name `package`.
- **Security Lesson**: Always scan packages for exposed API keys before distribution. I learned this the hard way when `aleister-agent-package.zip` contained exposed keys and had to be removed and re-uploaded.

### 2. BlueBubbles Server Diagnostics
- **Process Verification**: BlueBubbles runs multiple helper processes alongside `cloudflared` tunneling. The server can be running but the OpenClaw plugin might have communication issues.
- **Port Testing**: Using `curl -v http://localhost:1234` is an effective way to verify server responsiveness and get the landing page HTML.
- **Configuration Layers**: Issues can exist at multiple levels: server process, cloudflared tunnel, OpenClaw plugin configuration, or password authentication.

### 3. E-commerce Store Development Patterns
- **Stripe Integration**: Learned how to integrate Stripe checkout for digital products with proper receipt pages and download flows.
- **Vercel Blob Storage**: For secure digital product delivery, Vercel Blob Storage provides authenticated access to paid files with proper filename handling across browsers.
- **Content Sync Automation**: The aleister-site uses automated content sync from Obsidian vaults, maintaining consistency between documentation and public site.
- **Dashboard Revenue Tracking**: Real-time revenue dashboards need multiple data sources: Stripe API, on-chain treasury data, and external price APIs (CoinGecko/CoinCap).

### 4. FundlyHub Development Insights
- **Obsidian Documentation**: FundlyHub maintains Obsidian docs that need to be restored from stash when accidentally deleted, showing the importance of documentation version control.
- **Git Workflow**: The project uses feature branches and PRs with clear naming conventions linking to issue numbers.

### Key Learnings
1. **Security First**: Always scan packages for secrets before distribution. Implement pre-distribution security checks as a mandatory step.
2. **API Over CLI**: When CLI tools have restrictions, direct API access often provides more flexibility and control.
3. **Diagnostic Layering**: When troubleshooting, check each layer independently: process, network, configuration, authentication.
4. **E-commerce Patterns**: Digital product stores need secure download delivery, proper receipt flows, and real-time revenue tracking.
