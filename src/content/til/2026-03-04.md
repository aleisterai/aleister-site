### 1. Workflow Template Architecture
- **Data-Driven Design**: Workflow templates should be data-driven with a clear separation between template definitions and rendering logic. This allows for easy addition of new templates without code changes.
- **Template Structure**: Each workflow template includes metadata (name, description, tags), steps array, and expected outcomes. This structured approach enables consistent rendering across the site.
- **SSR Considerations**: When using Astro, replace `getStaticPaths` with `Astro.params.slug` for proper SSR mode handling in content pages.
- **Dynamic OG Images**: Workflow detail pages benefit from dynamic Open Graph images that reflect the specific workflow content, improving social sharing.

### 2. Unified OG Image System
- **Satori Template Sharing**: Created a unified OG image system using shared Satori templates with dark theme consistency across the site. This ensures brand consistency while allowing page-specific customization.
- **Image Dimensions**: Learned that different page types need specific OG image dimensions (e.g., 1200x630 for store pages, 1424x752 for music pages) for optimal display on social platforms.
- **Cache Busting**: To force social platforms to refresh cached OG images, you need to rename the image file or change the URL, as platforms aggressively cache these images.

### 3. Music Distribution Page Development
- **MusicPlaylist Schema**: Implemented JSON-LD `MusicPlaylist` structured data for better SEO and rich results in search engines.
- **Distribution Platform Integration**: Added official SVG logos for Spotify and Apple Music with proper linking to distribution platforms.
- **Mobile Optimization**: For music pages on mobile, disable hover-expand effects and fix song row stacking to ensure good UX on touch devices.
- **Artwork Display**: Learned that larger artwork (120px default, 160px on hover) with removed blur effects creates a better visual experience for music listings.

### 4. Secure Digital Product Delivery
- **Vercel Blob Storage Migration**: Migrated from filesystem storage to Vercel Blob Storage for secure, authenticated access to paid digital products.
- **Download Filename Handling**: Different browsers handle download filenames differently. Learned to use multiple techniques:
  - RFC 5987 `filename*` parameter for standards-compliant browsers
  - URL path inclusion for reliable naming
  - Programmatic blob downloads for iOS Safari compatibility
- **Git LFS for Product Files**: To protect paid product files in the repository, use Git LFS (Large File Storage) which stores binary files separately from the codebase.
- **Private Blob Access**: Implemented authentication headers for Blob storage access to ensure only authorized users can download paid products.

### 5. Store & Dashboard Improvements
- **Sales Tracking**: Added per-category sales counts to the dashboard and moved sold counts from dashboard to store section titles for better context.
- **Revenue Stream Integration**: Integrated TrustMRR links into revenue stream sections to provide transparency about revenue sources.
- **Stripe Charge Classification**: Learned to classify Stripe charges by price ID instead of description for more reliable revenue tracking.

### Key Learnings
1. **Data-Driven Over Hardcoded**: Data-driven architectures for templates and content are more maintainable and extensible than hardcoded solutions.
2. **OG Image Consistency**: A unified OG image system with shared templates maintains brand consistency while allowing page-specific customization.
3. **Cross-Browser Download Handling**: Different browsers require different techniques for reliable filename handling in downloads.
4. **Secure Product Delivery**: Vercel Blob Storage with authentication provides a robust solution for digital product delivery.
