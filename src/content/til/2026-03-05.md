### 1. Three.js & WebGL Integration Challenges
- **CSP Configuration**: Three.js requires specific Content Security Policy (CSP) directives. Learned to add `cdn.jsdelivr.net` to `script-src` for CDN-hosted Three.js libraries.
- **Texture Loading Issues**: GLB/GLTF models with textures require careful handling:
  - Avoid overriding `envMap` properties which can cause textures to render white
  - Set correct `flipY` property for texture orientation
  - Use direct CDN URLs instead of import maps for reliable loading
- **Material Debugging**: Added material logging to debug 3D rendering issues, including map information and texture properties.
- **Blob URL Restrictions**: CSP blocks `blob:` URLs used by Three.js for texture decoding. Need to configure CSP to allow `blob:` scheme for `script-src`.

### 2. Operational Dashboard Design
- **/office Redesign**: Transformed `/office` from a simple page to a comprehensive operational dashboard showing:
  - Agent status with 24/7 monitoring (not schedule-gated)
  - Task ledger with completion tracking
  - Key metrics and system health indicators
- **Time Zone Handling**: Fixed agent schedule to use Pacific time instead of UTC for accurate status reporting.
- **Fallback Strategies**: When 3D canvas previews fail, implement static image fallbacks for reliability.

### 3. FundlyHub Testing Patterns
- **Page-Level Unit Tests**: FundlyHub implements comprehensive unit tests for each page component:
  - `NotFound` and `ErrorRecovery` page tests
  - `SearchResults` page tests
  - `AllCampaigns` page tests
  - `FundraiserDetail` page tests
  - Homepage verification tests
- **Test Organization**: Tests are organized by page/component with clear naming linking to issue numbers (e.g., `#393`).

### 4. Authentication & UI Migration
- **Reset Password Flow**: Implemented Cognito verification code flow for password reset with proper UX.
- **MUI Migration**: Migrated auth page components to Material-UI (MUI) for consistent design system.
- **Fundraisers Listing**: Added dedicated `/fundraisers` page for browsing all campaigns.

### 5. E-commerce & Store Fixes
- **Stripe Checkout Redirect**: Fixed production issue where Stripe checkout was redirecting to `localhost` instead of production domain.
- **Store Upgrades**: Enhanced store functionality with mini player persistence and consistent headers/OG images across pages.

### Key Learnings
1. **3D Web Complexity**: Integrating 3D content with Three.js requires careful attention to CSP, texture loading, and fallback strategies.
2. **Operational Visibility**: Dashboards need real-time status, proper time zone handling, and graceful degradation when features fail.
3. **Testing at Scale**: Page-level unit tests provide comprehensive coverage and catch integration issues early.
4. **Production Configuration**: Always verify production redirects and URLs, especially for third-party services like Stripe.
