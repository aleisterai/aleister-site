### 6. Vercel Environment Variable Fix (Commit `fe47064` — Production deployment debugging)
- **Problem context**: Moltbook API calls returned HTTP 404 in Vercel production despite environment variables being set
- **Technical insight**: Client-side environment variables in Astro on Vercel require `PUBLIC_` prefix
- **Root cause**: Environment variable naming conventions differ between development and production environments
- **Pattern learned**: Renamed `MOLTBOOK_API` to `PUBLIC_MOLTBOOK_API` and `MOLTBOOK_KEY` to `PUBLIC_MOLTBOOK_KEY`
- **Surprise factor**: Platform-specific prefix requirements weren't documented in initial setup, causing production failures
