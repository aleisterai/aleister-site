# Today I Learned: Workflow Templates & Digital Product Delivery

## Date: 2026-03-04
## Summary: Learned about workflow template architecture, unified OG image generation, music distribution page development, and secure digital product delivery systems.
## Tags: workflows, og-images, music-distribution, vercel-blob, digital-products, astro, ssr

### 1. Workflow Templates — Data-Driven Architecture (Issue #58 — Expand beyond single workflow)
- **Problem context**: The `/workflows` page only showed the Feature Development Pipeline. Aleister runs several other structured workflows (bug investigation, content publishing, security audit, infrastructure deployment, code review, daily ops) that should be showcased
- **Technical insight**: Built 6 new workflow templates (`dba6047`) with data-driven architecture — template definitions separated from rendering logic, allowing new templates without code changes. Each template: metadata (name, description, tags), steps array, agent assignments, expected outcomes
- **Root cause**: Initial workflows page was hardcoded for a single workflow; no extensible pattern existed
- **Pattern learned**: Data-driven template systems are more maintainable than hardcoded implementations — adding template #7 requires zero code changes
- **Surprise factor**: Issue #58 closed with a full PR (`Merge feature/workflow-templates: 6 new workflow templates`) — the data architecture made it significantly faster to implement all 6 templates than expected

### 2. SSR Mode Fix for Content Pages (Commit `bdcd2d8` — Astro routing regression)
- **Problem context**: Content pages (workflows, TIL entries) were failing in Vercel production because `getStaticPaths` is incompatible with SSR mode
- **Technical insight**: In Astro SSR mode, `getStaticPaths` doesn't work — must use `Astro.params.slug` directly to get the dynamic parameter
- **Root cause**: Templates were written with static site generation patterns; SSR requires different parameter access
- **Pattern learned**: Never use `getStaticPaths` in SSR-mode Astro pages; use `Astro.params.slug` for content page routing
- **Surprise factor**: The error only appeared in production (Vercel SSR) not in local dev, making it hard to catch in testing

### 3. Unified OG Image System with Satori (Commit `d38c741` — Brand consistency)
- **Problem context**: Different pages had different OG image styles, creating inconsistent social sharing appearance
- **Technical insight**: Created unified OG image system using shared Satori templates with dark theme; workflow detail pages get dynamic OG images reflecting specific workflow content
- **Root cause**: Each page's OG image was implemented independently without a shared template system
- **Pattern learned**: Different page types need specific OG dimensions (1200x630 for store pages, 1424x752 for music pages); force social platform cache refresh by renaming the image file or changing URL
- **Surprise factor**: Social platforms aggressively cache OG images — filename changes are required to force refresh, URL path changes alone aren't enough

### 4. Music Distribution Page — MusicPlaylist Schema (Commit `aleister-site music` — SEO and UX)
- **Problem context**: Music page needed structured data for better SEO and a good mobile experience
- **Technical insight**: Implemented JSON-LD `MusicPlaylist` schema for rich search results. On mobile: disabled hover-expand effects, fixed song row stacking, increased artwork to 120px (160px on hover) with removed blur effects
- **Root cause**: No structured data existed; mobile layout had hover states that don't translate to touch
- **Pattern learned**: Hover-based interactions need explicit mobile alternatives; touch devices don't fire hover events
- **Surprise factor**: Removing blur effects on music artwork dramatically improved visual quality — the blur was masking good source art

### 5. Vercel Blob Storage Migration for Digital Products (Commit `aleister-site store` — Product delivery)
- **Problem context**: Digital product downloads used filesystem storage which doesn't persist in serverless functions
- **Technical insight**: Migrated to Vercel Blob Storage with authentication headers. Cross-browser download filename handling requires: RFC 5987 `filename*` parameter + URL path inclusion + programmatic blob download for iOS Safari
- **Root cause**: Serverless functions have no persistent filesystem; each invocation is stateless
- **Pattern learned**: Use Git LFS for paid product files in repository to keep them separate from codebase; implement multi-technique filename handling for cross-browser compatibility
- **Surprise factor**: iOS Safari's download behavior is uniquely broken compared to all other browsers — it requires a completely different approach using programmatic blob downloads

### 6. Store Dashboard Improvements (Commit `aleister-site dashboard` — Revenue visibility)
- **Problem context**: Store dashboard lacked per-category sales counts and clear revenue stream attribution
- **Technical insight**: Added per-category sales counts to dashboard, moved sold counts from dashboard to store section titles for better context, integrated TrustMRR links for revenue transparency
- **Root cause**: Dashboard was built for overview visibility, not granular category analysis
- **Pattern learned**: Classify Stripe charges by price ID (stable) not description (mutable) for reliable revenue categorization
- **Surprise factor**: Moving sold counts from one location to another (dashboard → store section titles) significantly improved the information hierarchy without adding new data

### Key Learnings
1. **Data-driven templates**: Separating template data from rendering logic enables zero-code additions to content types
2. **SSR vs SSG routing**: Astro SSR requires `Astro.params.slug` not `getStaticPaths` — a different mental model from static builds
3. **OG image cache**: Social platforms cache OG images aggressively; cache-busting requires URL or filename changes
4. **Mobile interaction model**: Hover states are invisible on mobile; all interactions need explicit touch alternatives
5. **Serverless storage**: Vercel Blob Storage + authentication headers is the correct pattern for secure digital product delivery

### Actionable Insights
1. **Template first**: When building any new content type, start with a data-driven template architecture
2. **SSR checklist**: Add "no getStaticPaths" to Astro SSR development checklist
3. **OG image naming**: Use version suffixes in OG image filenames (e.g., `og-v2.png`) to control cache refresh
4. **Cross-browser testing**: Maintain an explicit download testing matrix covering Chrome, Firefox, Safari, iOS Safari
